XML-RPC Tester

Como otra forma adicional de acceder al servidor, está la implementación en Javascript de un cliente de XML-RPC. Esta es otra muestra de la versatilidad que la utilización de este mecanismo ofrece. Además, dado que XML-RPC se sostiene en un servidor HTTP, es posible hacer que éste mismo servidor sirva (valga la redundancia) esta misma página, con el código javascript que hace de cliente de ese mismo. En resumen, dondequiera que se este ejecutando este servidor, ofreciendo los métodos oportunos de la librería, es posible acceder a sus servicios vía un navegador web con el javascript activado. Claro está, no deja de ser también un servidor XML-RPC en toda regla, con lo que cualquier otro cliente (por ejemplo, el basado en Python) puede conectarse al mismo.
Dado que, en este momento, la implementacion cliente en javascript no es mucho más que una prueba de concepto, no soporta más que llamadas simples a los métodos. Es decir:

zAdd( getRandomZ('100'), '123' )
no está soportado actualmente. Por contra, por ejemplo,
zAdd( '10', '123' )
sí lo está.

Los argumentos de las funciones son cadenas de caraceres (strings), que deben especificarse como en los ejemplos anteriores, entrecomilladas. Esto es también una consecuencia de que se están invocando los métodos de forma directa, sin capa intermedia que lime todas estas asperezas.

Para obtener una lista de las funciones soportadas por el servidor, hacer click en el botón correspondiente. NB: éste es un mecanismo automático: no es necesario cambio alguno en el cliente si a posteriori el servidor implementase más métodos o cambiase la especificación de los presentes. El cliente inspecciona dinámicamente qué es lo que ofrece el servidor.
